Requisitos del dataset

Los datos se encuentran en el dominio de las ciudades inteligentes

Los datos se basan en la ciudad de Zaragoza, dándonos la relación de incidencias (cortes de agua, cortes de tráfico y otras afecciones importantes) en las calles o viales de la ciudad.
http://www.zaragoza.es/ciudad/viapublica/movilidad/trafico/listado_Incidencia

Se dispone de datos como un archivo CSV

Nos dan un CSV pero solo con los 50 primeros registros, ya que para trabajar con el resto hay que hacerlo mediante consultas a la api.

Los datos tienen una licencia abierta para que pueda ser publicada

En los avisos legales dicen: "El usuario se compromete a hacer un uso de la plataforma adecuado con su finalidad participativa y a no emplearlo para incurrir en actividades ilícitas; contrarias a la buena fe; que vulneren o transgredan el honor, la intimidad personal o familiar o la imagen de las personas; o que supongan actos de competencia desleal o publicidad ilícita."
Por lo que se pueden utlizar siempre y cuando se hagan buen uso de ellos.

Los datos pueden ser fácilmente vinculados con entidades genéricas del mundo real (por ejemplo, ubicaciones)

Los datos son ubicables, ya que nos indican la direccion de la calle dónde se ha producido dicha incidencia.

Existe documentación de los datos (opcional)

Existe una api donde se puede hacer las consultas necesarias para sacar más información y una descripción interna dentro de cada dato.

Los datos pueden provenir de múltiples fuentes de datos (opcional)

Los datos provienen del Ayuntamiento de Zaragoza. Unidad de Gestión de la Web Municipal, cuya pagina principal de obtencion de los datos es: https://www.zaragoza.es/ciudad/risp/detalle_Risp?id=67
